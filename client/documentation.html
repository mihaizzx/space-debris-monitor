🌍 Prezentare Generală

Space Debris Tracker este o aplicație web avansată pentru monitorizarea și analiza deșeurilor spațiale în orbita Pământului. Utilizează date reale de la NASA și algoritmi de propagare orbitală SGP4 pentru a oferi o vizualizare 3D precisă a sateliților și deșeurilor spațiale.

🎯 Scop: Să ofere oricui acces la date critice despre deșeurile spațiale și să ajute la înțelegerea riscurilor asociate cu obiectele orbitale.

🚀 Tehnologii Utilizate

🌐 Frontend

  • Cesium.js 1.122 - Vizualizare 3D
  • JavaScript ES6+ - Logică aplicație
  • HTML5/CSS3 - Interfață modernă

🔧 Backend

  • FastAPI (Python) - REST API
  • Skyfield - Propagare SGP4
  • NumPy - Calcule științifice

📊 Surse de Date

  • CelesTrak - Date TLE
  • NASA ORDEM - Flux deșeuri
  • Space-Track.org - Catalog obiecte

    • Number of satellites: Câți sateliți să afișeze (5-100)

    • Click stânga + drag: Rotește globul

🔗 API-uri NASA Utilizate

1. 🛰️ CelesTrak TLE Data API

Scop: Furnizează Two-Line Element (TLE) sets pentru toate obiectele orbitale urmărite.

GET https://celestrak.org/NORAD/elements/gp.php?GROUP={group}&FORMAT=tle

Parametri:

  • GROUP - Categoria de sateliți (active, weather, communications, etc.)
  • FORMAT - Format răspuns (tle, json, xml)

Răspuns: Text format TLE (3 linii per satelit: nume, linia 1, linia 2)

📚 TLE Format:
Linia 1: Număr catalog, clasificare, epoch, derivate mișcare
Linia 2: Înclinație, ascensiune dreaptă, excentricitate, argument perigeu, anomalie medie, mișcare medie

2. 🌌 NASA ORDEM (Orbital Debris Engineering Model)

Scop: Model statistic pentru fluxul de deșeuri spațiale în funcție de altitudine și dimensiune.

# Model încorporat în aplicație (nu API extern) # Date preluate din NASA ORDEM 3.1 Technical Report

Parametri Model:

  • altitude_km - Altitudinea orbitei (200-50000 km)
  • inclination_deg - Înclinația orbitală (0-180°)
  • size_min_cm - Dimensiune minimă deșeu
  • size_max_cm - Dimensiune maximă deșeu

Output: Flux (impacts/m²/year) pentru fiecare interval de dimensiuni

🎯 Zone de Risc Identificate:
  • 400 km - LEO inferior (ISS, Hubble) - Flux: 1.2e-4 impacts/m²/an
  • 800 km - LEO mijlociu (Iridium) - Flux: 5.8e-5 impacts/m²/an
  • 1400 km - LEO superior - Flux: 3.2e-5 impacts/m²/an
  • 20200 km - MEO (GPS, Galileo) - Flux: 1.5e-6 impacts/m²/an
  • 35786 km - GEO (Comunicații) - Flux: 8.7e-7 impacts/m²/an

3. ☀️ NASA DONKI (Space Weather Database)

Scop: Informații despre evenimente solare și impactul lor asupra Pământului.

GET https://api.nasa.gov/DONKI/CME?startDate={date}&api_key={key} GET https://api.nasa.gov/DONKI/FLR?startDate={date}&api_key={key} GET https://api.nasa.gov/DONKI/GST?startDate={date}&api_key={key}

Endpoint-uri:

  • /CME - Coronal Mass Ejections (ejecții masă coronală)
  • /FLR - Solar Flares (erupții solare)
  • /GST - Geomagnetic Storms (furtuni geomagnetice)
  • /IPS - Interplanetary Shocks

Răspuns: JSON cu data, intensitate, durata și impact previzionat

⚠️ Impact asupra sateliților:
  • CME puternice pot deteriora electronica sateliților
  • Solar flares cauzează interferențe radio și GPS
  • Furtuni geomagnetice cresc frecarea atmosferică → decay orbital

4. 🌍 Space-Track.org API

Scop: Catalog oficial US Space Command cu date precise despre toate obiectele spațiale.

POST https://www.space-track.org/ajaxauth/login GET https://www.space-track.org/basicspacedata/query/class/gp/NORAD_CAT_ID/{id}

Autentificare: Necesită cont gratuit + login cu credențiale

Date disponibile:

  • General Perturbations (GP) - TLE data
  • Satellite Catalog - Metadata complete
  • Decay predictions - Predicții de reintrare
  • Launch site - Informații lansare

5. 🔭 Skyfield (SGP4 Propagator)

Scop: Librărie Python pentru calcule astronomice precise, inclusiv propagare SGP4.

from skyfield.api import EarthSatellite, load, wgs84, utc ts = load.timescale() sat = EarthSatellite(line1, line2, name, ts) geocentric = sat.at(t)

Funcționalități:

  • SGP4 Propagation: Calcul poziții satelit din TLE
  • Coordinate Transformations: Geocentric → Geodetic (lat/lon/alt)
  • Time Scales: UTC, TAI, TT pentru sincronizare precisă
  • Visibility: Calcul when/where satellites are visible

📐 Calcule Matematice și Formule

1. 🛰️ SGP4 Orbital Propagation (Simplified General Perturbations)

Algoritmul SGP4 propagă poziția satelitului din date TLE, luând în considerare perturbațiile gravitaționale și atmosferice.

Poziție satelit la timp t: r(t) = [x(t), y(t), z(t)] # Vector poziție 3D Calculat prin: 1. Extragere parametri orbitali din TLE (a, e, i, Ω, ω, M₀) 2. Calcul anomalie medie M(t) = M₀ + n·(t - t₀) 3. Rezolvare ecuație Kepler: M = E - e·sin(E) 4. Conversie la coordonate carteziene: x = a·(cos(E) - e) y = a·√(1 - e²)·sin(E) 5. Rotații pentru înclinație și ascensiune dreaptă 6. Ajustări pentru perturbații (J₂, J₃, J₄, drag atmosferic) Parametri TLE: • a = semi-major axis (km) • e = eccentricity (0-1) • i = inclination (degrees) • Ω = right ascension of ascending node (degrees) • ω = argument of perigee (degrees) • M₀ = mean anomaly at epoch (degrees) • n = mean motion (revolutions/day)

2. 🎯 Collision Probability Calculation

Probabilitatea ca un satelit să colizioneze cu deșeuri spațiale într-o perioadă dată.

Probabilitate anuală de coliziune: P_annual = 1 - exp(-F × A × T) Unde: • F = flux deșeuri (impacts/m²/year) [din NASA ORDEM] • A = suprafața satelitului (m²) • T = perioadă timp (years) Flux ORDEM simplificat: F(h, i, s) = F₀ × f_altitude(h) × f_inclination(i) × f_size(s) Unde: • h = altitude (km) • i = inclination (degrees) • s = debris size (cm) Factor altitudine (empiric): f_altitude(h) = { exp(-|h - 400|/200) dacă h < 1000 km (LEO) exp(-|h - 800|/300) dacă 1000 < h < 2000 km 1 / (h/1000)² dacă h > 2000 km (MEO/GEO) } Factor înclinație: f_inclination(i) = 1 + 0.3 × sin(2 × i × π/180) # Risc maxim la ~52° și ~98° (orbite polare/retrogade) Exemplu numeric: ISS: A = 10 m², h = 400 km, i = 51.6°, T = 1 year F ≈ 1.2e-4 impacts/m²/year (din ORDEM) P_annual = 1 - exp(-1.2e-4 × 10 × 1) ≈ 0.0012 = 0.12%

3. 📏 3D Distance Calculation (Euclidean Space)

Distanța în spațiu 3D între satelit și deșeu folosind coordonate carteziene.

Conversie coordonate sferice → carteziene: Pentru un punct la (lat, lon, alt): R = R_earth + alt # Rază totală de la centrul Pământului x = R × cos(lat) × cos(lon) y = R × cos(lat) × sin(lon) z = R × sin(lat) Unde R_earth = 6371 km (raza medie a Pământului) Distanța euclidiană: d = √[(x₁ - x₂)² + (y₁ - y₂)² + (z₁ - z₂)²] Exemplu numeric: Satelit 1: lat=0°, lon=0°, alt=400 km → x₁ = 6771 km, y₁ = 0 km, z₁ = 0 km Debris: lat=0°, lon=5°, alt=420 km → x₂ ≈ 6751 km, y₂ ≈ 590 km, z₂ = 0 km d = √[(6771-6751)² + (0-590)² + 0²] = √[400 + 348100] ≈ 590.3 km

4. 🌍 Orbital Velocity Calculation

Viteza orbitală a unui satelit la o altitudine dată.

Formula vitezei orbitale circulare: v = √(μ / r) Unde: • μ = GM_earth = 398600 km³/s² (parametru gravitațional standard) • r = R_earth + altitude (raza orbitală totală) Pentru orbite eliptice: v = √[μ × (2/r - 1/a)] Unde: • a = semi-major axis • r = raza curentă Viteza relativă între două obiecte: v_rel = |v₁ - v₂| (aproximativ, pentru orbite apropiate) v_rel_real = √[(v₁ₓ - v₂ₓ)² + (v₁ᵧ - v₂ᵧ)² + (v₁ᵤ - v₂ᵤ)²] Exemple numerice: ISS (h = 400 km): v = √(398600 / 6771) ≈ 7.67 km/s ≈ 27,600 km/h GPS (h = 20,200 km): v = √(398600 / 26571) ≈ 3.87 km/s ≈ 13,900 km/h GEO (h = 35,786 km): v = √(398600 / 42157) ≈ 3.07 km/s ≈ 11,000 km/h

5. ⏱️ Orbital Period (Perioada Orbitală)

Timpul necesar unui satelit pentru o revoluție completă.

Formula lui Kepler pentru perioada orbitală: T = 2π × √(a³ / μ) Unde: • T = perioadă (secunde) • a = semi-major axis (km) • μ = 398600 km³/s² Simplificat pentru orbite circulare: T = 2π × √(r³ / μ) Unde r = R_earth + altitude Număr de orbite pe zi: n = 86400 / T (revolutions/day) Exemple numerice: ISS (h = 400 km, r = 6771 km): T = 2π × √(6771³ / 398600) ≈ 5531 s ≈ 92.2 min n ≈ 15.6 orbite/zi GPS (h = 20,200 km, r = 26571 km): T = 2π × √(26571³ / 398600) ≈ 43,082 s ≈ 11.97 ore n ≈ 2 orbite/zi GEO (h = 35,786 km, r = 42157 km): T = 2π × √(42157³ / 398600) ≈ 86,164 s ≈ 23.93 ore n ≈ 1 orbită/zi (sincron cu rotația Pământului!)

6. 🎢 Risk Factor Calculation (Factorul de Risc)

Combinarea distanței și vitezei relative pentru evaluarea riscului de coliziune.

Factor de risc combinat: Risk = w_dist × f_distance + w_vel × f_velocity Unde: • w_dist = 0.7 (pondere distanță) • w_vel = 0.3 (pondere viteză) Factor distanță normalizat: f_distance = max(0, (d_max - d) / d_max) Unde: • d = distanța curentă (km) • d_max = raza maximă de căutare (ex: 2000 km) Valori: 1.0 (foarte aproape) → 0.0 (departe) Factor viteză normalizat: f_velocity = min(1, v_rel / v_max) Unde: • v_rel = viteza relativă (km/s) • v_max = 10 km/s (viteză maximă tipică) Valori: 0.0 (stationar) → 1.0 (impact frontal maxim) Interpretare Risk: • Risk > 0.8: 🔴 CRITICAL (risc extrem de mare) • 0.6 < Risk ≤ 0.8: 🟠 HIGH (risc mare) • 0.4 < Risk ≤ 0.6: 🟡 MODERATE (risc mediu) • 0.2 < Risk ≤ 0.4: 🟢 LOW (risc scăzut) • Risk ≤ 0.2: ⚪ MINIMAL (risc neglijabil) Exemplu numeric: Debris la d = 25 km, v_rel = 12 km/s, d_max = 2000 km: f_distance = (2000 - 25) / 2000 = 0.9875 f_velocity = min(1, 12/10) = 1.0 Risk = 0.7 × 0.9875 + 0.3 × 1.0 = 0.691 + 0.3 = 0.991 → 🔴 CRITICAL!
